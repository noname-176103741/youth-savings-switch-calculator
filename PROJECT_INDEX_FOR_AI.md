# 청년도약계좌 to 청년미래적금 계산 페이지 인덱스

## 프로젝트 목적

청년도약계좌를 2026-07-27 특별중도해지하고 2026-07-28 청년미래적금으로 전환하는 경우가, 청년도약계좌를 만기까지 유지하는 경우보다 우위인지 비교하는 GitHub Pages용 정적 계산 페이지다.

## 현재 완료 상태

- `index.html` 기반 단일 페이지 앱 구현 완료.
- 계산 로직은 `src/calc.js`에 분리되어 있고, UI는 `src/app.js`에서 연결한다.
- 전국은행연합회 청년미래적금 은행별 우대조건 스냅샷은 `src/data.js`에 정적 데이터로 포함되어 있다. 선택 은행과 우대금리 확인 탭 모두 요약, 상세조건, 만기 후 금리, 유의사항을 표시한다.
- 브라우저 `localStorage`로 설정값, 선택 은행, 활성 탭, 납입 이력, 수동 수정 여부를 자동 저장/복원한다.
- 청년도약계좌 유지 시 만기 원금 이자와 기여금 이자는 1~36개월, 37~48개월, 49~60개월 금리 스케줄로 나눠 단리 일할 계산한다. 기존 `youthLeapMaturityRate`와 `youthLeapContributionRate`는 1~36개월 금리로 유지하고, 4년차/5년차 입력 필드를 추가했다.
- `사용된 논리` 탭에서 현재 계산에 적용된 납입일, 한도, 기여금, 금리 스케줄, 외부 현금, 손익분기 규칙을 확인할 수 있다.
- Node 단위 테스트는 `tests/calc.test.mjs`에 있다.

## 먼저 읽을 파일 순서

1. `SUMMARY_FOR_ME.md`: 사용자가 보는 짧은 현재 상태.
2. `src/calc.js`: 날짜/납입/기여금/외부 현금 시뮬레이션의 핵심 로직.
3. `src/app.js`: DOM 입력, 납입 이력 편집, 결과 렌더링.
4. `src/data.js`: 공식 기준값과 은행별 우대금리 스냅샷.
5. `tests/calc.test.mjs`: 계산 규칙 검증 범위.

## 주요 파일 역할

- `index.html`: 앱 레이아웃, 결과 요약, 탭 패널, 입력 폼, 납입 이력 표.
- `src/styles.css`: 반응형 대시보드 스타일. 모바일에서 표는 가로 스크롤.
- `src/calc.js`: 순수 계산 모듈. 브라우저와 Node 테스트가 함께 사용. `calculateSegmentedSimpleInterest`가 기간별 금리 스케줄을 일수별로 나눠 이자를 계산한다.
- `src/data.js`: 청년도약계좌/청년미래적금 기준 데이터와 은행별 우대조건. KFB 검색 응답의 상세 우대조건 `<br>` 텍스트를 줄바꿈 문자열로 정리해 넣었다.
- `tests/calc.test.mjs`: 가입월 납입일, 2월 29일, 한도 검증, 기여금, 세후 이율, 기본 시나리오 테스트.
- `package.json`: `npm.cmd test` 스크립트만 둔 최소 설정.
- `.gitignore`: `temp/`, `.playwright-mcp/`, `node_modules/` 제외.

## 계산 구현 요약

- 청년도약계좌
  - 월한도 700,000원, 연한도 8,400,000원 검증.
  - 연한도는 달력연도가 아니라 가입일 기준 납입연도(`2024-06-29~2025-06-28` 같은 구간)로 검증.
  - 매년 가입월은 `max(입금일, 해당 연도 가입기념일)`로 자동 생성. 예: 2024-06-29 가입, 입금일 1일이면 2025-06 납입은 2025-06-29.
  - 해지일이 해당 월 납입 가능일보다 빠르면 그 달 납입은 생성하지 않는다. 예: 2024-07-29 가입, 2026-07-27 해지면 2026년 7월 납입 없음.
  - 납입 이력을 수동 수정하기 전에는 가입일/해지일/입금일/기본금액 변경 시 UI 납입 이력이 자동 재생성된다. 수동 수정 후에는 자동 덮어쓰기를 멈추며, 사용자가 `납입 이력 자동 생성` 버튼으로 초기화할 수 있다.
  - 사용자가 수정한 납입 이력은 `localStorage`에 저장되어 새로고침 후 복원된다. `저장값 초기화` 버튼은 저장값과 화면 상태를 기본값으로 되돌린 뒤 기본 상태를 다시 저장한다.
  - 2025-01 납입분부터 확대 기여금 적용.
  - 기여금은 다음달 10일 정산으로 처리하고, 정산일이 지급일 이후면 기여금은 포함하되 이자는 0원.
  - 유지 시 만기 지급액의 원금 이자는 가입일부터 36개월까지 1~36개월 금리, 36~48개월은 4년차 변동금리 예상값, 48~60개월은 5년차 변동금리 예상값으로 나눠 계산한다.
  - 유지 시 기여금 이자도 정산일부터 만기일까지 같은 3구간 금리 스케줄로 나눠 계산한다.
  - 특별중도해지 지급액은 해지일까지의 납입 이력을 기준으로 계산.
  - 특별중도해지 원금 이자는 아직 단일 입력금리 `youthLeapSpecialRate`로 계산한다.
  - 유지 시 만기일은 가입일 + 60개월.
- 청년미래적금
  - 월납입은 `min(설정 월납입액, 500,000원)`.
  - 만기일은 가입일 + 36개월.
  - 기여금 유형: 미지급, 일반형 6%/월 30,000원 한도, 우대형 12%/월 60,000원 한도.
- 외부 돈
  - 사용자는 세전 연이율 입력.
  - 계산은 `세전 x (1 - 0.154)` 세후 연이율을 `(1 + 세후연이율)^(1/365) - 1`로 일이율 환산.
  - 매일 이벤트 반영 후 음수 잔액이 있으면 두 시나리오에 같은 금액을 추가 투입한다.
- 손익분기
  - 외부 세전 연이율 -20%~30% 범위에서 부호 변화를 찾고 이분법으로 계산.

## 실행/테스트 명령

```powershell
npm.cmd test
```

현재 단위 테스트는 12개이며, 기간별 금리 스케줄 이자 계산 테스트가 포함되어 있다.

정적 서버 예시:

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

그 뒤 브라우저에서 `http://127.0.0.1:4173/` 열기.

## 공식 데이터 출처

- 청년도약계좌: `https://fill4young.kinfa.or.kr/yla/main`
- 청년미래적금: `https://ylaccount.kinfa.or.kr/`
- 은행별 우대금리: `https://portal.kfb.or.kr/compare/receiving_youth_future_2.php`
- 은행별 데이터 기준일: 2026. 6. 19.

## 실패했던 접근과 주의사항

- `node --test`는 기본 샌드박스에서 자식 프로세스 생성 `spawn EPERM`으로 실패했다. 권한 밖 실행에서는 정상 통과했다.
- KFB 페이지는 표를 정적 HTML 본문에 직접 싣지 않고 `/compare/receiving_youth_future_search_result.php` POST 응답으로 채운다.
- 현재 `src/data.js`의 은행별 우대조건은 `receiving_youth_future_2.php`에서 은행 체크박스 값을 읽고, `/compare/receiving_youth_future_search_result.php`에 `ordertype=''`, `str=<은행값|...>`로 POST해 받은 응답을 파싱한 것이다. 임시 원문과 파싱 결과는 `temp/kfb_youth_future_page.html`, `temp/kfb_youth_future_result.html`, `temp/kfb_youth_future_banks.json`에 있다.
- 은행별 우대조건은 공시 시차가 있을 수 있으므로, 공개 전 최신 데이터 갱신이 필요하면 `src/data.js`를 다시 갱신해야 한다.
